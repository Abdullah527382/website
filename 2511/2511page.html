Some Notes:

  • In java, array is an object (essentially everything is), placing a dot afterwards will show us the methods that a certain object has.

  • Unlike in C, we don't have to declare size of array, rather since array is an object, it will automatically increase in size as we add more.

Objected Oriented Programming in Java

As opposed to procedural programming e.g. C where functions are grouped to form programs, OOP does the following:

  • Uses and creates own user-defined types called classes
  • Classes contain data and methods (procedures)
  • An instance of a class is an object, which has attributes (data) and behaviors (methods) encapsulated into it.

Designing a class:
Whenever we want to create a class, we should consider a few things: The classes' methods, keeping data private, always initializing data, breaking up classes with too many responsibilities, etc.

Objects:
Are created by instantiating a class, example below:

Circle c = new Circle ( ) ;
double a;
// Then access the data:

// and radius 1.
// Assuming, x, y and r are not private
c.x = 2;
c.y = 5;
c.r = 1;

a = c.area( );

Super classes, Objects, and the Class Hierarchy
Every class has a superclass, by default if superclass isn't defined then it is the class Object

Often we'll need to override the following methods: toString(), equals(), hasCode()

Abstract classes:
These are classes which define only a part of an implementation i.e. these extended classes have some methods from the superclass. This is done so that the programmer knows the interface definition of an object.

  • An abstract class cannot be instantiated whereas it's subclass can if it overrides each of its superclass' abstract methods (but also provides an implementation for all of them, if not then that subclass is also abstract)

Interfaces in Java
These are similar to abstract classes where all methods defined within the interface are implicitly abstract.

  • Variables defined in an interface must be static and final i.e. constants
  • Classes can implement an interface (more than 1 actually), to do this; a class uses the implements keyword and then provides implementation for all abstract methods of the interface
  • Example below
// interface
public interface Animal {
public void animalSound(); // interface method (does not have a body)
public void sleep(); // interface method (does not have a body)
}

// Pig "implements" the Animal interface
class Pig implements Animal {
public void animalSound() {
  // The body of animalSound() is provided here
  System.out.println("The pig says: wee wee");
}
public void sleep() {
  // The body of sleep() is provided here
  System.out.println("Zzz");
}
}

class MyMainClass {
public static void main(String[] args) {
  Pig myPig = new Pig();  // Create a Pig object
  myPig.animalSound();
  myPig.sleep();
}
}

Extending Interfaces
Interfaces can have sub-interfaces just like classes can have subclasses

  • A sub-interface inherits all abstract methods and constants of its super-interface.

Method Overriding and Polymorphism
Polymorphism:

  • An object’s ability to decide what method to apply to itself, depending on where it is in the inheritance hierarchy, is usually called polymorphism.

When a class defines a method using the same name, return type, and by the number, type, and position of its arguments as a method in its superclass,
the method in the class overrides the method in the superclass. This is the new definition of the method that is invoked for an object of the class, and NOT the superclass' definition.
Note:

  • We may refer to the overridden method using super keyword.
  • We cannot call super.super.someMethod -> Not legal
class Animal {
 public void move() {
    System.out.println("Animals can move");
 }
}

class Dog extends Animal {
  // Method overriding
 public void move() {
    System.out.println("Dogs can walk and run");
 }
 public void bark() {
    System.out.println("Dogs can bark");
 }
}

public class TestDog {

 public static void main(String args[]) {
    Animal a = new Animal();   // Animal reference and object
    Animal b = new Dog();   // Animal reference but Dog object

    a.move();   // runs the method in Animal class
    b.move();   // runs the method in Dog class
    b.bark();
 }
}

This will produce errors for b.bark() since there is no such bark method in class Animal and b is of type Animal

Method Overloading
Defining methods with the same name and different argument or return types is called method overloading. A method signature consists of - its name, return type, number, type and position of its argumens, e.g.

double add(int, int)
double add(int, double)
double add(float, int)
double add(int, int, int)
double add(int, double, int)

Data Hiding and Encapsulation
We can hide the data within the class and make it available only through the methods. This is done through Visibility Modifiers of which Java has 5 access modifiers:

  • public -> visible to the world
  • private -> visible to the class only
  • protected -> visible to the package and all subclasses
  • No modifier (default) -> visible to the package

Constructors

A constructor in Java is a special method that is used to initialize objects. The constructor is called when an object of a class is created. It can be used to set initial values for object attributes:

  • If the first statement in a constructor is not a call to super() or this(), a call to super () is implicitly inserted.
// Create a MyClass class
public class MyClass {
int x;  // Create a class attribute

// Create a class constructor for the MyClass class
public MyClass() {
  x = 5;  // Set the initial value for the class attribute x
}

public static void main(String[] args) {
  MyClass myObj = new MyClass(); // Create an object of class MyClass (This will call the constructor)
  System.out.println(myObj.x); // Print the value of x
}
}

// Outputs 5
